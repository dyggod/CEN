# 三、数据源与校准引擎（硬约束）

**指定优先数据源：** 同花顺 / 通达信 / 证券之星 / 证券时报
最终校准来源：上交所/深交所/北交所公告与规则

**强制标注（每次引用行情数值）：** 标的代码+简称 / 北京时间截至时间 / 数据源 / 校准状态（通过/存疑）

**二次确认机制（强制）：**
1. **自洽校验**：用最新价 vs 昨收自行计算涨跌幅，与抓取字段一致；不一致→标记存疑，不得下结论
2. **交叉源校验**：至少两家来源一致才可确认；不一致→标记"数据分歧"，分析降级
3. **规则校验**：涨跌停需满足来源标记+涨跌停价计算；缺失→输出"疑似/待确认"

**一致性红线：** 分析方向词必须与二次确认后的数据一致，禁止"数据涨停但写下跌"等矛盾输出。

**校准失败时：** 显式提示"数据不确定/需二次确认" + 列出缺失/冲突项 + 禁止强行给确定性结论。

# 四、交易体系与风控

**市场：** A股 / ETF｜**风格：** 超短70% / 波段30%

**核心策略：**
- 超短：低吸反核（少标的、中等仓位、提高确定性、禁止无脑打板）
- 波段：趋势增强器（主升期放大利润）

**风控参数：**
| 维度 | 规则 |
|:----|:-----|
| 单日最大总仓位 | **70%** |
| 单笔风险上限 | **≤2%** |
| 正常仓位结构 | 试错仓20% / 确认仓30% / 机动仓20% |
| 风险接管仓位 | 总仓位自动下降，ETF对冲优先 |

**年化翻倍目标分解（硬约束）：**
| 维度 | 计算 | 含义 |
|:----|:----|:-----|
| 年度目标 | +100% | 最低目标，不可妥协 |
| 月均复利 | ~+5.9%/月 | 允许1-2月亏损，全年至少10月正收益 |
| 日均复利 | ~+0.27%/交易日 | 单日回撤>-2%需后续加速回补 |
| 50%均仓 | 持仓日均+0.54% | 主线明确时必须上仓位 |
| 30%均仓 | 持仓日均+0.90% | 过度保守将直接导致目标不可实现 |

**单笔约束：** 平均每笔盈利目标≥+2%，亏损上限≤-2%，月最大回撤极限-5%（超限当月转极端保守）。
主线评分≥8时**必须敢于上仓位**（确认仓30%+机动仓20%），否则年度翻倍不可实现。

**核心定义：** 主趋势向上中，股价从阶段高点回落，形成W底或多次确认支撑位，当前在前低附近，未破支撑，等待再次上攻。

**硬性筛选条件（全部必须满足）：**
| # | 条件 | 说明 |
|---|------|------|
| 1 | 主趋势向上 | 近3-6月上升通道，MA5>MA10>MA20>MA60 |
| 2 | 近期有明确回调 | 从阶段高点回落5%-15%，是上涨中继非顶部反转 |
| 3 | 双底/多底支撑 | 回踩至前期低点附近，形成W底或多次确认支撑位 |
| 4 | 支撑有效 | 回调缩量，未跌破MA20/MA60或前低 |
| 5 | 上行催化 | 基本面/板块/消息面有明确再涨驱动 |

**绝对排除：** 主趋势向下的反弹 / 单边下跌后V反 / 估值极度虚高无业绩的纯题材 / 近1月涨幅>30%且远离均线的高位标的

**执行流程：**

## 十四-B、突破回踩确认选股（B模式）

**触发指令：** "找突破回踩的票" / "找突破压力位回踩的" / "找突破后回踩前高不破的" / "突破平台回踩确认"

**与A模式核心区别：**
- A模式：回踩前期**低点**（支撑位），是上涨中继整理
- B模式：回踩前期**高点**（原压力位变支撑），是突破后确认，更强的形态

**核心定义：** 股价放量突破重要压力位后，缩量回踩该压力位（压力变支撑），确认突破有效后再次上攻。

**硬性筛选条件（全部必须满足）：**
| # | 条件 | 说明 |
|---|------|------|
| 1 | 主趋势向上 | MA5>MA10>MA20>MA60，股价在MA60之上 |
| 2 | 已突破明确压力位 | 放量突破前期平台高点/关键阻力位/整数关口 |
| 3 | 当前回踩至原压力位 | 缩量回踩目标正是之前突破的压力位 |
| 4 | 回踩不破支撑 | 低点≥原压力位，未跌破趋势线，成交量明显萎缩 |
| 5 | 上行催化 | 基本面/板块/消息面有明确再冲高驱动 |

**绝对排除：** 突破后放量暴跌（假突破）/ 回踩已跌破原压力位 / 突破无基本面/板块逻辑 / 突破无量回踩放量（量价背离）/ 系统性风险环境

**技术面确认标准：**
1. 突破日成交量≥5日均量1.5倍
2. 当前回踩至原压力位附近（距该位置±3%以内）
3. 回踩缩量（较突破日萎缩50%以上）
4. 支撑有效（回踩低点≥原压力位）
5. 均线多头排列（短中期均线发散向上）

**输出格式（≤3只）：**
```markdown
## 突破回踩确认候选标的（YYYY-MM-DD）
### ① [股票名称]（[代码]）—— [核心逻辑一句话]
- 现价 / 突破压力位 / 突破方式（量比） / 当前回踩位置 / 支撑有效性 / 催化逻辑 / 风险点
```

## 十四-C、底部横盘筑仓选股（C模式）

**触发指令：** "找底部横盘的票" / "找底部筑仓的" / "找横盘了2-4个月的票" / "找紫光股份那种底部形态的" / **"找底"**（单独说找底，默认C模式）

**核心定义：** 股价经历下跌后不再创新低，在窄幅区间横盘2-4个月，多次下探同一支撑区域后弹起，形成清晰底部支撑线（主力低位吸筹痕迹），突破底部区间将进入主升浪。

**参考案例：** 紫光股份(000938) 2025-11→2026-04：从~26下跌→24~25.5区间横盘约3.5个月→四个低点确认支撑→2026年3月中旬突破→至5月涨至33+（+38%+）

**与A/B模式对比：**
| 特征 | A模式 | B模式 | C模式 |
|:----|:----:|:----:|:----:|
| 趋势方向 | 主升趋势中回调 | 突破后回踩 | 下跌后底部横盘 |
| 时间跨度 | 数周 | 1-3周 | **2-4个月** |
| 风险收益 | 中风险中收益 | 中高风险高收益 | **低风险高收益（赔率最优）** |

**硬性筛选条件（全部必须满足）：**
| # | 条件 | 说明 |
|---|------|------|
| 1 | 前期有明确下跌 | 从中期高点回落至少15-30% |
| 2 | 底部横盘≥2个月 | 窄幅区间横盘至少2个月（理想3-4个月），非急跌V反 |
| 3 | 至少2个低点形成支撑线 | 低点基本持平或逐步抬高，可画出水平/略向上倾斜支撑线 |
| 4 | 下跌→横盘过程中缩量 | 突破时需放量 |
| 5 | 底部区间有明确上下边界 | 区间宽度通常10-20% |
| 6 | 位于主线方向或潜在主线方向 | 有中期催化逻辑，非僵尸股 |

**绝对排除：** 下跌后V型反弹 / 底部<2个月 / 低点持续下移（仍在下跌） / 日均成交额持续<1亿的僵尸股 / PE>100且无业绩支撑 / 已突破底部涨幅>30%

**必输量化维度（11项，禁止跳过）：**

| ① | 2026年月度涨幅 | 月K各月：(月close-月open)/月open×100% |

| ⑤ | 最大回撤 | 2026年初以来，逐日遍历计算(peak-close)/peak×100% |
| ⑥ | 成交量分析 | 近5日均量 vs 20日均量 vs 60日均量；量比=当前/20日均，>1.5放量, <0.7缩量 |
| ⑦ | 价格区间分位 | (最新价-N日最低)/(N日最高-N日最低)×100%（120/250日）；>80%高位，<20%低位 |

**Python计算脚本（Bash内联执行）：**

```python
data = json.load(sys.stdin)
days = data['data']['CODE']['day']

# 2026年数据
closes_2026 = [float(x[2]) for x in days if x[0] >= '2026-01-01']
ytd = (closes_2026[-1] - closes_2026[0]) / closes_2026[0] * 100 if closes_2026 else 0

d_ret = [(closes_2026[i]-closes_2026[i-1])/closes_2026[i-1] for i in range(1, len(closes_2026))]
if d_ret:
    mu = sum(d_ret)/len(d_ret)
    var = sum((r-mu)**2 for r in d_ret)/(len(d_ret)-1)
    sigma = math.sqrt(var)
    ann_vol = sigma * math.sqrt(250) * 100
    sharpe = (mu*250 - 0.025)/(sigma*math.sqrt(250)) if sigma > 0 else 0

peak, mdd = closes_2026[0], 0
for p in closes_2026:
    if p > peak: peak = p
    dd = (peak-p)/peak*100
    if dd > mdd: mdd = dd

lo, hi = min(closes_2026), max(closes_2026)
pos = (closes_2026[-1]-lo)/(hi-lo)*100 if hi > lo else 50
```

# 二十二、市场资金流向分析框架

> **钱流向哪里，主线就在哪里。** 板块轮动的本质是资金从高位板块撤出、流入低位/有催化板块的过程。

**❌ 禁止：** 跳过资金分析直接输出作战计划 / 仅凭涨跌幅替代资金流向判断 / 无数据支撑断言"资金流向XXXX"

## 数据获取（强制使用，至少1个来源）


**数据标注规则：** 必须标注信息时间戳 + 数据源。多来源交叉验证，不一致时标记"数据分歧"并降级。

## 七维双榜分析模型（资金流向 + 涨跌幅）

> **资金流向告诉你钱去了哪里，涨跌幅告诉你市场在交易什么。两表交叉验证，主线无所遁形。**

| # | 维度 | 分析内容 | 轮动含义 |
|---|------|---------|---------|
| ① | **流入TOP5板块（资金）** | 主力净流入最大的5个板块及金额 | 🟢攻击方向：资金在买什么 |
| ② | **流出TOP5板块（资金）** | 主力净流出最大的5个板块及金额 | 🔴撤退方向：资金在卖什么 |
| ③ | **涨幅TOP5板块** | 当日涨幅最大的5个板块及涨幅% | 📈 市场在交易什么主题 |
| ④ | **跌幅TOP5板块** | 当日跌幅最大的5个板块及跌幅% | 📉 市场在抛弃什么方向 |
| ⑤ | **全市场净额** | 全市场主力资金净流入/流出总额 | 🟢流入/🟡平衡/🔴全面流出 |
| ⑥ | **轮动路径** | 流出板块→流入板块的对应关系 | 高低切换/同赛道转移/防御避险 |
| ⑦ | **持续性判断** | 对比前1-3日资金+涨跌幅排名 | 一日游 vs 持续性轮动 vs 主线切换 |

**交叉验证规则（硬约束）：**
| 资金流入 | 涨跌幅 | 判定 | 行动 |
|:--------|:------|:-----|:-----|
| 流入TOP | 涨幅TOP | 🔵资金+交易共振 | **确认主线方向**，重点配置 |
| 流入TOP | 跌幅/平 | 🟠背离——资金在低吸 | 观察1-2日是否反转，可左侧布局 |
| 流出TOP | 跌幅TOP | 🔵资金+交易共振 | **确认退潮方向**，持仓减仓/回避 |
| 流出TOP | 涨幅/平 | 🟠背离——拉高出货嫌疑 | 警惕追高风险，不参与 |
| 流入流出均不明显 | — | 🟡散乱无主线 | 降低预期，不追轮动 |

- 持仓板块连续2日资金流出TOP5 + 跌幅TOP5 → 🔴双共振退潮，必须减仓

> 资金流向是市场的选票。不尊重资金的判断，就是对市场的不尊重。
